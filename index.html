HTML

  • Enter your HTML notes here.
  • HTML creates the basic structure of a webpage. It is made up of elements that work together to provide content and a basic framework for all pages on the web.
  • Elements can be types of content on a page (text, images, headers, lists), or they can be used to divide and describe the content on the page to browsers, other developers, and accessibility technology. These would be meta-related elements, content division elements, and semantic elements.
  • The html opening tag represents the root of the HTML document, and it's closing tag represents the end of the document.
  • The head element contains information about the webpage.
  • The title element defines the title of a webpage and is very important to Search Engine Optimization.
  • The body element represents the visible content shown to the user.
  • The heading element displays headings larger and bolder, semantically signifying importance in code. They are ordered h1-h6, most significant to least.
  • Lists can be formatted in HTML with the ordered list element (ol) and the unordered list element (ul). Both have list item elements (li).

Git

  • Enter your Git notes here
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: add a file with changes to stage them for committing
  • git add -A: commits all files in the working branch
  • git commit -m "": commit a staged file with a detailed, brief commit message tracking the change made to the code
  • git push origin branch-name: pushes commits from the local repository to github
