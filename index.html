HTML

  • Enter your HTML notes here:
  • HTML creates the basic structure of a webpage. It is made up of elements that work together to provide content and a basic framework for all pages on the web.
  • Elements can be types of content on a page (text, images, headers, lists), or they can be used to divide and describe the content on the page to browsers, other developers, and accessibility technology. These would be meta-related elements, content division elements, and semantic elements.
  • The html opening tag represents the root of the HTML document, and it's closing tag represents the end of the document.
  • The head element contains information about the webpage.
  • The title element defines the title of a webpage and is very important to Search Engine Optimization.
  • The body element represents the visible content shown to the user.
  • The heading element displays headings larger and bolder, semantically signifying importance in code. They are ordered h1-h6, most significant to least.
  • Lists can be formatted in HTML with the ordered list element (ol) and the unordered list element (ul). Both have list item elements (li).

CSS

  • Enter your CSS notes here:
  • Cascading Style Sheets (CSS): A programming language used by developers to define how webpage content is presented to users, defining colors, aesthetic, visual layout, and organization.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External CSS sheets are the most common and best practice. CSS is kept in a separate file linked to the html page, which makes design changes easier.
  • Separation of concerns: A widely used term in web development and refers to separating code into different files based on use. It follows the belief that each section of code should have its own responsibility.
  • Do Not Repeat Yourself (DRY): A coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code. CSS selectors can share stylings to keep code from becoming too repetitive.

Git

  • Enter your Git notes here:
  • Git is an open source code management and version control system tool. We use it with GitHub, an online platform that pairs with Git to store, track, and aid collaboration with software projects.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add: add a file with changes to stage them for committing
  • git add -A: commits all files in the working branch
  • git commit -m "": commit a staged file with a detailed, brief commit message tracking the change made to the code
  • git push origin branch-name: pushes commits from the local repository to github

JavaScript

  • Enter your JavaScript notes here:
  • A high-level dynamic programming language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code.
  • An array is a single variable that is used to hold a group of data. The data is typically related in some way.
  • An array index indicates the position of the element within the array. The first item is at index 0 and they follow a sequential order after that (0,1,2,3).
  • Control flow is the order in which a computer executes a code in a script. It executes linearly, top to bottom.
  • Conditionals are code structures used to test if an expressions remains true or not, if a conditon is met.
  • For loops perform a task on all items in an array by allowing a single code block to be executed over and over.
  • Functions are a set of defined instructions within a code block that tell a computer to do a task and have a unique name. The defined function can then be called in our code as often as we want to, and the function will not execute until it is called.
